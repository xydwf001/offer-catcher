# 「Offer 捕手」方案说明

## 问题诊断

学生求职的主要困难不是岗位数量不足，而是筛选成本高、匹配标准不透明、简历优化缺少具体反馈。很多同学只能按岗位名称海投，无法判断岗位 JD 与自己项目、技能和兴趣的真实重合度，也不知道简历初筛更看重哪些关键词和证据。

## 方案设计

Offer 捕手将流程拆成四步：学生画像输入、实时岗位拉取、简历命中率诊断、投递行动生成。Demo 接入 Remotive 官方公开远程岗位 API，按用户目标方向和能力关键词拉取真实岗位，再计算岗位匹配分，并展示技能覆盖、方向贴合、岗位来源、发布时间和原始岗位链接。选择岗位后，系统继续计算初筛命中率，输出缺失关键词、项目证据不足点、可改写简历句子和投递优先级。

## AI 工具选型理由

Demo 使用公开岗位 API + 前端规则引擎。Remotive API 无需密钥，适合部署在 GitHub Pages；前端规则引擎负责岗位标准化、技能抽取、岗位排序和简历命中率拆解，优点是评分逻辑透明、可解释、可实时刷新。真实产品可继续接入大模型完成简历语义解析、JD 深度抽取和个性化话术生成。

## 关键配置

项目为静态 HTML/CSS/JS，部署在 GitHub Pages。页面根目录包含 `index.html`，发布方式选择仓库 `main` 分支的 `/` 根目录。关键配置包括 Remotive API 请求、6 小时本地缓存、请求超时处理和原始岗位链接跳转。接口失败时只显示错误与重试入口，不展示编造岗位。

## 迭代记录与效果评估

第一版完成岗位匹配、简历评分、差距解释和复制建议；第二版优化移动端布局和视觉层级；第三版移除网站内方案 PDF 入口，并接入实时岗位源。演示中，用户刷新岗位或修改简历关键词后可看到岗位顺序和命中率变化，说明系统能把“该投什么”和“简历怎么改”转成可执行反馈。
