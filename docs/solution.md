# 「Offer 捕手」方案说明

## 问题诊断

学生求职的主要困难不是岗位数量不足，而是筛选成本高、匹配标准不透明、简历优化缺少具体反馈。很多同学只能按岗位名称海投，无法判断岗位 JD 与自己项目、技能和兴趣的真实重合度，也不知道简历初筛更看重哪些关键词和证据。

## 方案设计

Offer 捕手将流程拆成四步：学生画像输入、官方岗位更新、简历命中率诊断、投递行动生成。Demo 汇总腾讯、字节跳动、京东、网易、联想招聘官网岗位，按目标方向、国内城市和能力关键词计算匹配分；同时提供华为、阿里巴巴、美团、百度等官方入口。用户可按公司筛选，点击卡片进入对应官网职位页。

## AI 工具选型理由

Demo 使用公司官方招聘接口、官网页面快照和前端规则引擎。岗位先在更新脚本中标准化为同源 JSON，解决跨域和不同字段格式问题；前端负责技能抽取、公司/城市筛选、岗位排序和简历命中率拆解。真实产品可继续接入大模型完成语义解析和个性化改写。

## 关键配置

项目为静态 HTML/CSS/JS，部署在 GitHub Pages。更新器读取腾讯、京东、网易公开接口，解析联想国内岗位，并通过浏览器获取字节官网动态签名后的职位数据，最终生成 `data/china-jobs.json`。前端设置 6 小时缓存；单个来源失败时保留其他真实岗位，不生成编造内容。

## 迭代记录与效果评估

第一版完成岗位匹配、简历评分和差距解释；第二版优化移动端布局并移除网站内 PDF 入口；第三版从单一岗位源升级为多家中国大厂官网数据，加入公司筛选和分批展示。当前快照含 300 余个真实岗位，修改画像、城市或简历关键词后，岗位顺序与命中率会同步变化。
