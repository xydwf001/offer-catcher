# 「Offer 捕手」方案说明

## 问题诊断

学生求职的主要困难不是岗位数量不足，而是筛选成本高、匹配标准不透明、简历优化缺少具体反馈。很多同学只能按岗位名称海投，无法判断岗位 JD 与自己项目、技能和兴趣的真实重合度，也不知道简历初筛更看重哪些关键词和证据。

## 方案设计

Offer 捕手将流程拆成四步：学生画像输入、官方岗位更新、简历命中率诊断、投递行动生成。Demo 定时读取腾讯招聘官网的校园与实习岗位，按目标方向、地区偏好和能力关键词计算匹配分；同时提供字节跳动、阿里巴巴、美团、百度、京东、华为的官方校招入口。页面展示职位、地点、分类、更新时间和官方链接，点击岗位可进入腾讯官网或 Workday 正式申请页。

## AI 工具选型理由

Demo 使用腾讯招聘官方公开接口 + 前端规则引擎。由于腾讯接口不允许 GitHub Pages 浏览器跨域读取，项目通过定时更新脚本生成同源 JSON 快照，兼顾真实性与部署稳定性。前端负责技能抽取、岗位排序和简历命中率拆解；真实产品可继续接入大模型完成简历语义解析和个性化改写。

## 关键配置

项目为静态 HTML/CSS/JS，部署在 GitHub Pages。GitHub Actions 每 6 小时运行更新脚本，读取腾讯校园与实习岗位并生成同源 JSON 快照；前端设置 6 小时缓存并保留官方职位链接。更新失败时保留上次真实快照，不生成或展示编造岗位。

## 迭代记录与效果评估

第一版完成岗位匹配、简历评分、差距解释和复制建议；第二版优化移动端布局和视觉层级；第三版移除网站内 PDF 入口，改用中国大厂官方数据。当前快照含 104 个腾讯真实岗位，所有卡片均可进入官方职位页；修改画像或简历关键词后，岗位顺序和命中率会同步变化。
