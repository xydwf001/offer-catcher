# 「Offer 捕手」方案说明

## 问题诊断

学生求职的主要困难不是岗位数量不足，而是筛选成本高、匹配标准不透明、简历优化缺少具体反馈。很多同学只能按岗位名称海投，无法判断岗位 JD 与自己项目、技能和兴趣的真实重合度，也不知道简历初筛更看重哪些关键词和证据。

## 方案设计

Offer 捕手将流程拆成四步：学生画像输入、岗位智能匹配、简历命中率诊断、投递行动生成。Demo 内置数据分析、AI 产品、前端、运营等校招岗位样例，用户输入专业背景、目标方向、城市偏好和简历文本后，系统会计算岗位匹配分，并展示技能覆盖、兴趣一致性、城市适配、简历证据和竞争风险。选择岗位后，系统继续计算初筛命中率，输出缺失关键词、项目证据不足点、可改写简历句子和投递优先级。

## AI 工具选型理由

Demo 使用前端规则引擎模拟智能体工作流，优点是无需密钥、可公开部署、评分逻辑透明，适合作业演示。真实产品可接入大模型完成简历语义解析、JD 信息抽取和个性化话术生成；当前版本先用可解释规则保证稳定性，避免黑箱评分。

## 关键配置

项目为静态 HTML/CSS/JS，部署在 GitHub Pages。页面根目录包含 `index.html`，发布方式选择仓库 `main` 分支的 `/` 根目录，无需后端服务和密钥，适合公开演示与作业验收。

## 迭代记录与效果评估

第一版完成岗位匹配、简历评分、差距解释和复制建议；第二版优化移动端布局、空状态和无障碍标签；第三版补充方案说明 PDF 与部署工作流。演示中，用户修改简历关键词后可立即看到命中率变化，说明系统能把“该投什么”和“简历怎么改”转成可执行反馈。
